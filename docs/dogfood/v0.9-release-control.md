# v0.9 Release Control Dogfood

## Scenario

Verify the release-control loop end to end: deploy dispatch, observation, rollback-decision recording, and operator visibility, using the offline fake GitHub Actions API and optional live providers.

## Offline Readiness Path

`scripts\release-readiness.ps1 -Version 0.9.0` runs the `release control smoke` and `release rollback smoke` steps against a local `HttpListener` fake GitHub API. The fake server answers `POST .../dispatches` with `204` and `GET .../runs` with a completed, successful workflow run. No `GITHUB_TOKEN` is required for the default gate.

## Live Dogfood Instructions (Optional)

Live release control is opt-in and dispatches a real GitHub Actions workflow. Set a real token and run against a demand in the `deployment` state:

```powershell
$env:GITHUB_TOKEN = "<github token>"
devflow deploy trigger --demand <id> --provider github --github-repo "owner/repo" --workflow "release.yml" --ref "main"
devflow observe refresh --demand <id>
devflow status --demand <id>
```

Do not run live release control from the default readiness gate.
