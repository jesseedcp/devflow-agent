# v1.7 Page-Driven Lifecycle Test Case

## Purpose

Verify that a real operator can use the Web UI, backed by MySQL and the platform server, to create a workspace, create a demand, inspect artifacts, confirm a manual gate, and understand which actions are not yet page-backed.

## Environment

- MySQL reachable at `127.0.0.1:3316`.
- Backend:
  `devflow server --addr 127.0.0.1:8088 --root <repo-or-demo-root>`
- Frontend:
  `VITE_DEVFLOW_API_MODE=http`
  `VITE_DEVFLOW_API_BASE=http://127.0.0.1:8088/api`
  `npm run dev -- --host 127.0.0.1 --port 5183`

## Test Data

- Workspace ID: `ui-dogfood`
- Workspace name: `UI Dogfood`
- Demand key: `coupon-ui-dogfood`
- Demand title: `Coupon UI dogfood`
- Demand description: `Inactive users must be blocked from claiming coupons.`

## Steps

1. Open `http://127.0.0.1:5183`.
2. Go to Workspaces.
3. Click `创建工作区`.
4. Fill workspace ID, name, and artifact root.
5. Submit and verify the new workspace appears.
6. Open the workspace demands page.
7. Click `新建需求`.
8. Fill demand key, title, and description.
9. Submit and verify the demand detail page opens.
10. Verify artifact tabs show generated demand artifacts.
11. Verify quality and next actions are visible.
12. Click `通过下一步`.
13. Verify the state changes from `requirements_review` to `plan_drafting`.
14. Verify runtime-only actions are disabled with a clear reason.
15. Go to Release and Wiki pages.
16. Verify unbacked actions are disabled with clear copy instead of throwing.
17. Open Audit and verify workspace/demand actions were recorded.

## Pass Criteria

- No visible button throws or silently does nothing.
- Workspace creation works against the real backend.
- Demand creation works against the real backend.
- Demand detail is populated from backend lifecycle data.
- `通过下一步` advances a supported manual gate.
- Unsupported actions are disabled and explain why.
- Audit contains records for platform actions.
