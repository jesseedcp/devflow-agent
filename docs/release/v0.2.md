# Devflow v0.2 Platform Integrations

## Release Status

v0.2 focuses on platform intake and collaboration sync. Default release readiness remains offline and credential-free.

## What v0.2 Contains

- GitHub Issue demand intake (rejects pull requests that share the Issues API shape).
- GitHub Issue progress sync via marker-aware issue comments.
- Feishu Doc read-only PRD intake with block-to-Markdown rendering.
- Feishu Bitable demand pool listing, record intake, and status sync.
- Platform doctor checks for GitHub and Feishu credentials.

## Supported First-User Path

1. Run `devflow doctor --platform all`.
2. Run `devflow intake --github-issue owner/repo#123` or `devflow intake --feishu-doc <doc-token>`.
3. Run the existing requirements, plan, implementation, review, verification, and closeout stages.
4. Run `devflow sync --github-issue owner/repo#123 --demand <id>` or `devflow sync --feishu-bitable <app-token> --table <table-id> --record <record-id> --demand <id>`.

## Known Limits

- Feishu Doc intake preserves semantic text, not exact rich document layout.
- Feishu Bitable sync updates existing records only.
- Default release readiness uses fake servers and does not validate live platform permissions.
- Logs, monitoring, long-term wiki writes, codemap refresh, and review-comment automation remain outside v0.2.
