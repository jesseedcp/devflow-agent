# Devflow v0.3 Acceptance Evidence Automation

## Release Status

v0.3 turns acceptance evidence into a first-class, fetchable verification input.

## What v0.3 Contains

- Generic `EvidenceRecord` acceptance evidence model.
- Backward-compatible `devflow evidence add`.
- New `devflow evidence fetch`.
- HTTP/API evidence adapter with pass/fail/blocked classification.
- Link/report evidence adapter with reachability checks.
- Secret redaction before writing artifacts or events.
- `evaluate`, `status`, `console`, and `workbench` support generic acceptance evidence.
- Offline release-readiness smoke for API evidence fetch.

## Known Limits

- No SLS, Grafana, Datadog, or complex log query adapters.
- Link evidence checks reachability only.
- Evidence fetch does not auto-confirm verification.
- Evidence fetch does not publish releases or update wiki/codemap.

## Verification Before Release

- `go test ./... -count=1 -timeout 5m`
- `go vet ./...`
- `go build ./cmd/devflow`
- `git diff --check`
- `scripts\release-readiness.ps1 -Version 0.3.0`
