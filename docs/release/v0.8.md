# Devflow v0.8.0

## Summary

v0.8.0 adds local deterministic metrics for measuring agent effectiveness across demands.

## New Commands

```powershell
devflow metrics report
devflow metrics report --demand <id>
```

The report reads `events.jsonl` and demand metadata, renders a reviewable markdown summary, and writes `metrics.md` into the demand directory. Metrics also surface inline in `status`, `console`, and `workbench --snapshot` output, and the closeout evaluation warns when a metrics report is missing.

## What Is Measured

- Human confirmations
- Review returns (requirements, plan, implementation)
- Verification runs, passes, and failures
- Acceptance evidence pass/fail/blocked
- Wiki candidates distilled, promoted, and rejected
- CI gate pass/block events
- Demand duration from local metadata/events

## What Is Not Included

- No telemetry upload
- No external analytics backend
- No semantic quality score
- No dashboard or chart UI

## Verification

- go test ./... -count=1 -timeout 5m

- scripts\release-readiness.ps1 -Version 0.8.0

## Known Limits

- Metrics are advisory in v0.8; they do not block closeout automatically.
- The closeout metrics check is warning-level only.
- The TUI workbench model view does not render metrics counts; only `--snapshot` is covered.
