# Devflow v1.0.1

v1.0.1 hardens runtime dogfood behavior discovered during the local weather-agent end-to-end test.

## Highlights

- Runtime-generated artifacts are now validated before the workflow advances.
- Empty runtime output now fails with an actionable stage/model/iteration error.
- The plan prompt now requires `## 实施步骤`, matching the existing `plan.steps` evaluation check.
- The implementation prompt now requires exact changed files and exact verification commands.
- Completed workspace summaries now show recorded release evidence correctly, for example deployment `passed` and observation `passed` instead of `pending`.

## Operator Impact

If `devflow run --stage plan` or `devflow run --stage implementation` returns an incomplete artifact, Devflow now stops in the current state with a clear validation error. Operators can retry with a stronger provider, adjust the artifact manually, or inspect provider compatibility without accidentally advancing the workflow.

## Boundaries

- No provider-specific retry loop was added.
- No new workflow states were added.
- No external provider, wiki, observability, or deployment adapter behavior changed.
