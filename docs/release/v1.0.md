# Devflow v1.0.0

## Summary

Devflow v1.0.0 finalizes the backend business demand Agent platform after the v0.9 release-control loop. The project now covers the local end-to-end path from demand intake through requirements, planning, implementation, review, verification, deployment, observation, closeout, knowledge distillation, and metrics.

v1.0.0 is a completion boundary, not a new feature wave.

## Included Capability

- Demand intake from local PRD files, URLs, GitHub Issues, Feishu Docs, and Feishu Bitable records.
- Requirements, plan, implementation, MR/PR review, verification, deployment, observation, and closeout workflow stages.
- GitHub PR creation/review/CI support and GitLab compatibility paths.
- Acceptance evidence from manual records, HTTP/API fetches, and report links.
- Release-control loop with GitHub Actions deployment evidence, post-deployment observation, and human-gated rollback decisions.
- Local long-term wiki distillation through `wiki-candidates.md` and `.devflow/wiki`.
- Local Agent effectiveness metrics through `devflow metrics report` and `metrics.md`.
- Operator surfaces through `status`, `console`, `workbench --snapshot`, `evaluate`, `drive`, and release-readiness smokes.

## v1.0 Boundary

The v1.0 boundary is offline-verified and credential-free by default. Live provider checks remain opt-in so the release can be reproduced without private credentials.

## Known Limits

- GitHub Actions is the only deployment provider in the release-control loop.
- Observation uses release-provider evidence only; no SLS, Grafana, Datadog, Prometheus, or OpenTelemetry adapter is included.
- Feishu platform integration is fake-server verified by default; full live Feishu dogfood still requires disposable Feishu credentials and assets.
- External wiki providers such as Feishu Wiki, GitHub Wiki, Notion, and Confluence are out of scope.
- Metrics are local and advisory; there is no telemetry upload, external analytics backend, or dashboard UI.
- Rollback decisions are recorded, but rollback execution is not automated.
- Enterprise user/team/tenant permission modeling is not included.
