# Devflow v1.1.1

Devflow v1.1.1 hardens runtime completion after the v1.1 Ark `glm-5.2` live dogfood.

## Shipped

- Runtime runner records structured tool traces and completion summaries.
- Implementation max-iteration runs can finish through a disclosed deterministic `progress.md` finalizer when edits and passing tests are already observed.
- Empty, read-only, and failing-test max-iteration runs still fail.
- Runtime completion events are written to demand events.
- `devflow metrics report` includes runtime efficiency counts.
- Implementation prompt tells models to stop using tools and return `progress.md` after passing tests.

## Verification

- `go test ./... -count=1 -timeout 5m`
- `go vet ./...`
- `go build ./cmd/devflow`
- `git diff --check`
- `scripts\release-readiness.ps1 -Version 1.1.1`
- Live Ark `glm-5.2` runtime completion dogfood

## Known Limits

- The deterministic finalizer is a safety fallback, not proof that code is correct. Existing quality gates and implementation review remain authoritative.
- Runtime efficiency metrics are advisory in v1.1.1.
- Live provider dogfood still requires external credentials and quota.
