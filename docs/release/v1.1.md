# Devflow v1.1.0

Devflow v1.1.0 hardens the runtime provider loop after live dogfood with Ark `glm-5.2`.

## Shipped

- Runtime plan stages now allow safe read/search tool prompts while still denying mutation tools.
- Plan evaluation accepts valid implementation steps written under child headings.
- Runtime artifact validation rejects heading-only artifacts before advancing workflow state.
- OpenAI-compatible streamed tool calls have regression coverage for Ark-style fragmented arguments.
- Agent loop tests verify tool results are available to the next model turn.
- Runtime max-iteration errors include stage, model, tool count, and recent tool names.

## Verification

- `go test ./... -count=1 -timeout 5m`
- `go vet ./...`
- `go build ./cmd/devflow`
- `git diff --check`
- `scripts\release-readiness.ps1 -Version 1.1.0`
- Live runtime provider dogfood with Ark `glm-5.2`

## Known Limits

- v1.1 does not add a provider-specific Ark SDK; it continues through the OpenAI-compatible adapter.
- v1.1 does not auto-edit artifacts after failed evaluation; failed artifacts still require a corrected rerun or human review.
- Live dogfood requires an external API key and must not commit secrets.
