# Devflow v1.2.1

Devflow v1.2.1 hardens release observation diagnostics after the v1.2 live GitHub Actions dogfood.

## Shipped

- `devflow doctor --observation-url <url>` checks whether the CLI can reach a post-deploy observation URL.
- Doctor redacts sensitive query values and prints a proxy recovery hint when the URL is unreachable.
- `observe refresh --health-url` records timed-out or unreachable health checks as `blocked` evidence.
- Blocked health checks still write `rollback.md` and route to the release decision gate.
- Release-readiness includes an offline v1.2.1 observation hardening smoke.
- User docs explain `HTTP_PROXY`, `HTTPS_PROXY`, and `NO_PROXY` for environments where browsers use a proxy but Go CLI processes do not.

## Operator Note

Go HTTP clients read proxy settings from environment variables such as `HTTPS_PROXY`, `HTTP_PROXY`, and `NO_PROXY`. A URL working in a browser does not prove the Devflow CLI can reach it from the shell.

PowerShell example:

```powershell
$env:HTTPS_PROXY="http://127.0.0.1:21882"
$env:HTTP_PROXY="http://127.0.0.1:21882"
```

Then retry:

```powershell
devflow doctor --observation-url https://github.com
devflow observe refresh --demand <demand-id> --health-url <url>
```
