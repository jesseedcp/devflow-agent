# Devflow v1.2.0

Devflow v1.2.0 proves the release-control loop against a real GitHub Actions workflow and adds minimum post-deploy HTTP observation evidence.

## Shipped

- Safe `.github/workflows/release.yml` with `workflow_dispatch`.
- `deploy trigger --github-input key=value` to pass Devflow demand context into release workflows.
- GitHub Actions dispatch polling so Devflow records the newly dispatched run rather than an older run.
- `observe refresh --health-url` for post-deploy HTTP health evidence.
- `observation.md` provider checks with redacted health evidence.
- Offline release-readiness smoke for dispatch inputs, run polling, and HTTP observation.
- Live GitHub Actions dogfood report.

## Not Shipped

- Devflow does not execute production deployment scripts directly.
- Devflow does not execute rollback automatically.
- SLS, Grafana, Datadog, Prometheus, and Harness adapters remain future work.

## Verification

- `go test ./... -count=1 -timeout 5m`
- `go vet ./...`
- `go build ./cmd/devflow`
- `git diff --check`
- `scripts\release-readiness.ps1 -Version 1.2.0`
- Live `devflow deploy trigger` against `jesseedcp/devflow-agent` GitHub Actions `release.yml`
