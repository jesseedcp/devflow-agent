# v0.6.1 Cleanup And v0.7 Knowledge Distillation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` or `superpowers:executing-plans` to implement this plan task-by-task. Keep this checklist updated as each task is completed.

**Goal:** Ship a small `v0.6.1` hardening release that cleans stale development branches/worktrees and tightens implementation-review guidance, then ship `v0.7.0` with a local long-term wiki / knowledge distillation loop: `closeout -> wiki candidate -> review -> promote`.

**Architecture:** `v0.6.1` is cleanup plus hardening only. It must not add a new product workflow surface. `v0.7.0` adds a deterministic local wiki lane that reuses the existing demand artifact store, events log, closeout material, and `internal/memory` safety patterns. Wiki candidates remain human-reviewed until explicitly promoted or rejected. Promoted wiki entries live under `.devflow/wiki` and become searchable local knowledge, but no external wiki write-back is introduced.

**Tech Stack:** Go standard library only, existing CLI conventions under `internal/cli`, existing artifact store under `internal/artifacts`, existing demandflow evaluation patterns under `internal/demandflow`, existing memory promotion safety ideas under `internal/memory`, PowerShell release-readiness smokes. No new dependencies.

	Now            func() time.Time
}

	CandidateIndex int

}
