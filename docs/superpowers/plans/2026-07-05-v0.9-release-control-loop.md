# v0.9 Release Control Loop Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship v0.9.0 by adding the missing release-control loop between verified code and closeout: GitHub Actions deployment trigger/status, post-deploy observation, human-gated rollback decision, workspace visibility, release-readiness coverage, and user documentation.

**Architecture:** Add a provider-neutral release-control domain under `internal/releasecontrol`, backed first by a GitHub Actions adapter in `internal/adapters`. Persist release evidence as three first-class demand artifacts (`deployment.md`, `observation.md`, `rollback.md`) and release events in `events.jsonl`. Extend workflow from `verification -> closeout` to `verification -> deployment -> observation -> closeout`; keep rollback as a human decision gate, not an automatic execution path. Surface the loop in CLI, status, console/workbench summaries, evaluation checks, and offline release-readiness with fake GitHub endpoints.

**Tech Stack:** Go standard library, existing `internal/artifacts`, `internal/templates`, `internal/workflow`, `internal/adapters`, `internal/demandflow`, `internal/cli`, PowerShell `scripts/release-readiness.ps1`, GitHub REST API shape verified through `httptest`. No new dependencies.

Rendering rules:

- Always trim free-text fields before rendering.
- Default empty status to `not_started` for deployment/observation.
- Default empty rollback decision to `pending`.
- Render timestamps with `time.RFC3339` only when non-zero.
- Keep markdown stable and easy to parse; avoid tables.

Core helper shape:

```go
func statusOrDefault(status Status) Status {
	if strings.TrimSpace(string(status)) == "" {
		return StatusNotStarted
	}
	return status
}

func decisionOrDefault(decision RollbackDecision) RollbackDecision {
	if strings.TrimSpace(string(decision)) == "" {
		return RollbackPending
	}
	return decision
}
```

- [ ] Implement `internal/releasecontrol/parse.go`.

Parsing rules:

- Look for backtick metadata lines such as `Status: ` + "`passed`" + ``.
- Return zero-value records when artifacts are missing or template-only.
- Normalize status strings to known statuses; unknown non-empty values become `StatusUnknown`.
- Normalize rollback decisions to known decisions; unknown non-empty values become `RollbackPending`.

Core parser shape:

```go
func metadataValue(text, key string) string {
	for _, line := range strings.Split(text, "\n") {
		line = strings.TrimSpace(line)
		prefix := key + ":"
		if !strings.HasPrefix(line, prefix) {
			continue
		}
		value := strings.TrimSpace(strings.TrimPrefix(line, prefix))
		value = strings.Trim(value, "` ")
		return value
	}
	return ""
}
```

```go
func TestGitHubActionsDispatchFindsRun(t *testing.T) {
	var sawDispatch bool
	server := httptest.NewServer(http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
		switch {
		case r.Method == http.MethodPost && r.URL.Path == "/repos/owner/repo/actions/workflows/release.yml/dispatches":
			sawDispatch = true
			if got := r.Header.Get("Authorization"); got != "Bearer secret-token" {
				t.Fatalf("Authorization = %q", got)
			}
			w.WriteHeader(http.StatusNoContent)
		case r.Method == http.MethodGet && r.URL.Path == "/repos/owner/repo/actions/workflows/release.yml/runs":
			w.Header().Set("Content-Type", "application/json")
			fmt.Fprint(w, `{"workflow_runs":[{"id":123,"html_url":"https://example/runs/123","head_sha":"abc123","head_branch":"main","status":"completed","conclusion":"success","created_at":"2026-07-05T10:00:00Z","updated_at":"2026-07-05T10:02:00Z"}]}`)
		default:
			t.Fatalf("unexpected request %s %s", r.Method, r.URL.Path)
		}
	}))
	defer server.Close()

	result, err := (GitHubActionsAdapter{Client: server.Client()}).TriggerDeployment(context.Background(), DeploymentRef{
		Repo:       "owner/repo",
		WorkflowID: "release.yml",
		Ref:        "main",
		BaseURL:    server.URL,
		Token:      "secret-token",
	})
	if err != nil {
		t.Fatalf("TriggerDeployment returned error: %v", err)
	}
	if !sawDispatch {
		t.Fatal("dispatch endpoint was not called")
	}
	if result.Status != DeploymentStatusPassed || result.RunID != "123" || result.HeadSHA != "abc123" {
		t.Fatalf("result = %#v", result)
	}
```

```go
func normalizeGitHubActionsDeploymentStatus(status, conclusion string) DeploymentStatus {
	status = strings.ToLower(strings.TrimSpace(status))
	conclusion = strings.ToLower(strings.TrimSpace(conclusion))
	if status != "completed" {
		return DeploymentStatusPending
	}
	switch conclusion {
	case "success", "neutral", "skipped":
		return DeploymentStatusPassed
	case "failure", "cancelled", "timed_out", "action_required":
		return DeploymentStatusFailed
	default:
		return DeploymentStatusUnknown
	}
}
```

## Risk Notes

- The broadest risk is changing `verification -> closeout` into `verification -> deployment -> observation -> closeout`; existing tests that assume direct closeout will need careful updates.
- `drive` and `console --run-next` must stop before live deployment. A release trigger is a real side effect and must be explicit.
- Rollback wording must stay precise: v0.9 records decisions, it does not roll back systems.
- GitHub Actions dispatch returns `204 No Content`; run discovery must list recent workflow runs. Fake-server tests should lock this behavior.
- Default readiness must remain offline and credential-free.
